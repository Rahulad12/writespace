# Architecture Decision Records

An ADR is written every time a non-obvious architectural or technology decision is made.
Run `/adr new <title>` to create one — do not copy the template manually.

## When to write an ADR
- Choosing a library or framework over alternatives
- Designing a cross-package contract or API shape
- Deciding on a data model that will be hard to change
- Changing an existing pattern that affects multiple packages
- Any decision that caused discussion or uncertainty

See `ADR-000-template.md`.
